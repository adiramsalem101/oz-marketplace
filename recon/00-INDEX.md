## Headline findings

1. **The oz-marketplace repo is completely bare.** Zero commits, no Next.js scaffold, no Tailwind, no SCSS, no Supabase config. The rebuild starts from a blank `next create` — there is nothing to remove or migrate inside oz-marketplace itself. (`05-oz-marketplace-state.md`)
2. **Total docs file count:** 43 across `docs/` (13 root markdown + 30 brand-folder mixed). Top-level subdirectories: `docs/` and `docs/brand/`.
3. **Brand token count by category:** ~44 colors (7 brand + 4 variants + 1 legacy + 8 semantic + 5 fg + 3 borders + 12 grays + 4 status), 4 font families × 4 weights declared (only `-400-` files actually shipped), 9-step type scale, 5 font weights, 7 radii, 6 shadows, 3 transitions + 1 keyframe, ~50 icons. **Spacing, breakpoint, and z-index scales are NOT defined in `brand/`** — needs decisions before scaffolding.
4. **oz-marketplace styling stack as found:** Tailwind = NO. SCSS = NO. CSS = NO. CSS Modules = NO. There is nothing to remove. The brand system's `colors_and_type.css` + `styles.css` + `fonts/` can be ported directly into `app/globals.scss` + module files during scaffolding.
5. **Schema object count (legacy live):** 6 enums, 27 public tables, 7 functions, 8 triggers, 35+ RLS policies (plus 9 storage policies). Three doc-flagged "missing" tables (`property_documents`, `monthly_rentals`, `property_owners`) confirmed absent. All schema artifacts come from one canonical file: the 3293-line baseline dump.
6. **Integration count:** 16 external vendors referenced, of which 8 are live (Supabase, Vercel, Google OAuth, Twilio, HelloSign, Google Gemini in code, FrontDeskMaster externally, Pakal Nofesh as reference). 3 are planned-but-not-built (Pelecard, Resend, Anthropic per spec). 2 are banned (Stripe, Meshulam) — but Stripe sneakily appears as `hostel_bookings.stripe_session_id`. 1 is unknown (Neema — not found).
7. **Biggest open question:** **AI Import provider mismatch.** Every doc says Anthropic Haiku; the actual edge function code imports `@google/generative-ai` (Gemini). Decision needed before the AI Import feature is reimplemented. (`OQ-1` in `10-open-questions.md`)
8. **Most load-bearing iron rule for the rebuild:** **IRON_RULE 1 — schema isolation.** The hostel domain (`hostel_*`) and the marketplace/corporate domain must remain physically separated — no FKs across them. Confirmed enforced in the live schema (`hostel_bookings` has only one FK, to `auth.users`). (`09-hostel-booking-engine.md`)
9. **Hebrew/RTL is non-negotiable** (IRON_RULE 5). Tokens already encode this (`body { direction: rtl }`). The first scaffold action should set `<html lang="he" dir="rtl">` and `body { direction: rtl; font-family: Heebo, Assistant, ... }`. (`05-oz-marketplace-state.md` §6.6, `04d-brand-tokens-extracted.md`)
10. **Pelecard is the only allowed payment provider** (IRON_RULE 2). Implementation pattern: mirror Pakal Nofesh. AM HOSTELS clearing accounts will be reused. No Stripe, no Meshulam. (`08-integrations.md`)
